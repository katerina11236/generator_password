 Менеджер паролей (самохост): Генерация, шифрование; бэк — AES, 2FA.
Спека:

		Введение:

Разрабатываемое программное обеспечение представляет собой саморазмещаемый (self-hosted) менеджер паролей, предназначенный для безопасного хранения, генерации и управления учетными
данными как индивидуальным пользователям, так и организациями. В условиях роста числа киберугроз и участившихся случаев утечек данных из облачных хранилищ, использование локально 
развертываемого решения обеспечивает необходимый уровень контроля и защиты конфиденциальной информации.
 
		Обзор:

		Цель спецификации.

Обеспечение конфиденциальности и целостности хранимых учетных данных пользователя путем применения криптографических методов защиты информации,
благодаря основному международному симметричному алгоритму блочного шифрования - AES.
Данный метод широко используется по всему миру: от смартфонов до военных систем. При возможности перехвата файла с базой данных паролей с сервера,
злоумышленник не сможет прочитать ни одного пароля без мастер-пароля.

Применение метода двухфакторной аутентификации(2FA) для усиления защиты процесса аутентификации и предотвращения несанкционированного доступа 
к расшифрованным данным (даже при компрометации мастер-пароля).
Двухфакторная аутентификация требует подтверждения личности с использованием двух независимых факторов, что существенно снижает риски 
несанкционированного доступа.

Разработка API-приложения, для хранения и генерации паролей. Реализация в виде API-приложения позволяет централизовать логику безопасности на 
серверной стороне, обеспечивая единые стандарты шифрования и аутентификации для всех клиентских приложений.

		Объём проекта:

		Краткое описание проекта.

Необходимо разработать программу для хранения и генерации паролей, которую пользователь сможет установить на свой собственный сервер. 
Программа будет хранить все пароли в зашифрованном виде, чтобы никто посторонний не мог их прочитать, а также помогать пользователю создавать
сложные и надежные пароли. Для входа в программу потребуется не только пароль, но и дополнительное подтверждение через специальное приложение на 
телефоне (двухфакторная аутентификация). Также программа будет предоставлять программный интерфейс (API), чтобы другие программы могли с ней 
взаимодействовать.

		Общее описание

Программа предназначена для людей и компаний, которые хотят хранить свои пароли в безопасности, а также получать помощь в создании новых 
надёжных паролей. При этом пользователи полностью контролируют место хранения своих данных.

Как именно достигается цель, процесс работы 

1. Физические лица - обычные люди, у которых много аккаунтов в интернете. Им нужно место, где можно хранить все пароли и заходить с телефона или компьютера.
	Как обычный пользователь я хочу сохранять свои пароли в одном месте чтобы не держать их в голове и не записывать в блокнот (как он хочет это сделать)

2. Семьи - несколько человек, которым нужен общий доступ к домашним паролям (Wi-Fi, подписки на кино, умный дом). Каждый входит под своим логином, но видит общие пароли.
	Как член семьи	я хочу видеть общие пароли (Wi-Fi, подписки) чтобы не просить их каждый раз у других

3. Технически подкованные пользователи - люди, у которых дома стоит свой сервер. Они не хотят хранить данные в интернете и предпочитают всё контролировать сами.
	Как пользователь со своим сервером я хочу настроить регулярное резервное копирование чтобы не потерять пароли при сбое

4. Небольшие компании (до 50 сотрудников) - организации, где сотрудники работают с общими аккаунтами (соцсети, реклама, хостинги). Им нужно, чтобы пароли не передавали в мессенджерах и
было видно, кто и когда заходил.
	**Как сотрудник я хочу иметь доступ к общим паролям отдела чтобы выполнять свои рабочие задачи.

5. IT-компании и веб-студии - фирмы, где работают программисты. Им нужно, чтобы программа умела отдавать пароли по запросу другим программам (через API) для автоматической настройки
серверов.
	**Как разработчик я хочу хранить пароли от клиентских проектов отдельно чтобы случайно не перепутать доступы

6. Некоммерческие организации - разные клубы и объединения с небольшим бюджетом. Им нужна бесплатная программа для общего доступа к паролям.
	**Как координатор я хочу дать волонтёрам доступ к общим аккаунтам чтобы они могли работать с соцсетями и почтой**

7. DevOps и системные администраторы - специалисты, которые настраивают серверы. Им важно, чтобы можно было получать пароли автоматически через API и устанавливать программу
на своих серверах.
	**Как администратор я хочу получать пароли из хранилища в скриптах чтобы не хранить их в открытом виде в коде

Функционал будет реализован посредством следующих инструментов:

1. FastAPI
2. PostgreSQL
3. Redis — для быстрой работы и хранения сессий
4. Стандартные библиотеки AES-256
5. Библиотеки для генерации и проверки TOTP-кодов
6. React или Vue.js
7. Docker
 
		Требования к функционалу:
Приложение должно обеспечить пользователю следующие функции
	добавление: пользователь вводит название сайта, логин и пароль. Программа сохраняет это в зашифрованном виде.
	удаления пароля: можно удалить ненужную запись.
	просмотр паролей: пользователь видит список всех сохранённых записей. Пароль показывается только по запросу.
	редактирование: можно изменить любой сохранённый пароль или его описание
	поиск: можно быстро найти нужный пароль по названию сайта или логину
	категории: можно раскладывать пароли по папкам (например, "Работа", "Личное", "Банки")
	импорт и экспорт: можно перенести пароли из другой программы или сохранить их в файл (например, для резервной копии)
	создание случайного пароля	Программа сама придумывает пароль, который трудно взломать
	настройка сложности: пользователь может указать, какой длины должен быть пароль, нужны ли заглавные буквы, цифры и специальные символы
	проверка надёжности: программа оценивает, насколько сложный пароль придумал пользователь (слабый, средний, надёжный)
	копирование в буфер: сгенерированный пароль можно сразу скопировать, чтобы вставить на сайте
	вход в программу: пользователь вводит мастер-пароль
	двухфакторная аутентификация: после ввода пароля программа просит ввести код из приложения на телефоне
	настройка 2FA: пользователь может включить или отключить двухфакторную аутентификацию у себя в настройках
	выход из системы: программа закрывает доступ к паролям, если пользователь вышел из системы
	общий доступ: можно дать другому пользователю доступ к определённой папке или паролю
	права доступа: можно разрешить только смотреть пароль или разрешить редактировать
	получение списка паролей: другая программа может запросить список сохранённых записей
	получение конкретного пароля:	Можно запросить пароль для конкретного сайта
	создание записи: можно добавить новый пароль через запрос
	генерация пароля: можно попросить программу сгенерировать пароль и сразу его получить
